# Diagrams

Architecture diagrams live here. They use Mermaid for GitHub and Obsidian compatibility.

## Diagrams

- [System context](system-context.md) - actors, systems, and responsibility boundaries.
- [Azure resources](azure-resources.md) - deployed Azure services and relationships.
- [Runtime flow](runtime-flow.md) - MCP startup and tool discovery through APIM.
- [Delivery and operations](delivery-operations.md) - local development, GitHub, Terraform, and Azure operations.

## Diagram Guidance

Keep diagrams focused. Prefer several small diagrams over one large drawing:

- Context diagrams explain the "why" and major boundaries.
- Resource diagrams explain the "what exists".
- Runtime sequence diagrams explain "what happens when".
- Delivery diagrams explain "how change moves".
